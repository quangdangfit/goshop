# Testing conventions

- Tests use `testify/suite`. Each suite has a `SetupTest` that builds a real service/handler wired to mock dependencies.
- Mocks live in `{package}/mocks/` and are generated by [mockery](https://github.com/vektra/mockery). Config is in `.mockery.yml` at the repo root.
- Every interface that needs mocking has a `//go:generate mockery --name=InterfaceName` comment above it.
- Handler tests create a `*gin.Context` via `gin.CreateTestContext` and assert on the `httptest.ResponseRecorder`.
